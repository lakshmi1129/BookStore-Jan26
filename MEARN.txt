                            3. Array with value as state/props/variables as dependency: Side effects will apply at the time of component creation/ loaded to DOM and whenever the value in dependancy changes. it is similar to Updating phase.

          MongoDB
    -------------------------------------
    1. MongoDb is NOSQL, Document oriented DB, that store data in JSON like format(BSON) instead of table.
    2. Features 
        - Schema-less (Flexible Structure)
        - Scalable
        - Designed for Mordern app.
    3. Key features
        - Database - A container of collections
        - Collection - Equalent to table in SQL
        - Document - A single record(JSON like object)
        - BSON : Binery version of JSON used internally
        - _id Field : Evry document has a unique key-Value.
    4. CRUD operations
        - Read : To get document from collection
            - find(): used to get all documents, 
                    - usage : collection-name.find()
                    - to query from collection : find({condition as object})
            - findOne(condition) : used to get single document
        - Create : To insert document into the collection
            - insertOne(document) : used to insert one document into a collection, Document can be an object.
            - insertMany([document1,document2.....]) : used to insert multiple documents at a time into a collection.
        - Update :  To edit document from a collection
            - updateOne({filter},{field:{updateQuery:value}}) : used to update a single document
            - updateMany({filter},{$updateQuery:{field:value}}) : used to update multiple document at sametime
        - Deletion:  To remove document from collections    
            - deleteOne() ; used to remove single document 
            - deleteMany() : used to remove multiple documents
    5. Query operations 
        - Comparison : syntax: {field:{Comparison-Query:value}}
            - eg: $eq, $ne, $gt, $gte, $lt,$lte,$in,$exists -(age:{$exists:true})...
        - Logical : syntax: {logical-query:[{field1:value1},{field2:value2}]}
            - eg: $and,$or,$nor,$not
        - Evaluation : syntax: {field:{Evaluation-query:value}}
            - eg: $regex,$text,$where
    6. Additional Oprations : count(), sort(),limit()
    7. Update operations :
        - Field : $inc,$set, $unset,$rename,$currentDate
        - Array : $push,$pop,$addToSet,$pull
    8. Indexing: used to make data searchimg faster in MongoDB
        - Why? - Faster find( queries, sorting, Imporve search by fields, Better performance for large data.)
        - Syntax: db.collection.createIndex({fieldName: 1})
    9. Schema design : Structure of collection.
        - Embedding Schema Design : add entire document details to it.
        - Referenced Schema Design  : using primary and foreign key for reference
    10. Aggregation: used to process and tranform data inside db, It is used for reports, analytics, filtering, grouping and sorting.
        - Syntax: db.collection.aggregate([{stage1},{stage2}......])
        - MongoDB uses pipeline stages : $match, $group, $sort,$project,$limit,$skip,$lookup
        - $lookup to join collections,  syntax:
                                            db.collection.aggregate([
                                                { $lookup: {
                                                    from: "From collection data is accessed", 
                                                    localField : "local collection field",
                                                    foreignField :"Foreign collection field", 
                                                    as :"new field name"}}])
    11. MongoDB Atlas is a cloud based Database-as-a-service (DBaas) provided by mongodb. It allows developers to deploy, manage and scale mongoDB databses on the cloud without handling  server setup or maintainance.
        - key Features of Atlas: Cloud Hosting, Automatic scaling, Backup and recovery, Monitoring, easy connection.
